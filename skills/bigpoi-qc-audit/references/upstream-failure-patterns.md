# Upstream Failure Patterns

## Usage

当上游结论看似有证据，但你怀疑“证据和结论不是一回事”时，逐条检查以下模式。命中时，把模式名称写入输出中的 `matched_failure_patterns`。

## Patterns

### FP01 同名不同地

- 表现：证据证明了某个同名 POI，但城市、区县、景区或交通枢纽不一致。
- 常见后果：名称被误判通过，存在性也被误判通过。
- 质检动作：先锁定行政区，再看名称。

### FP02 用旧名当现名

- 表现：证据来自旧公告、旧地图、旧媒体稿件。
- 常见后果：名称、类型、地址沿用过期信息。
- 质检动作：检查发布时间，确认是否已更名、搬迁、撤销或升级。

### FP03 把管理机构当成 POI 本体

- 表现：证据指向“管理委员会”“机场集团”“景区管理处”“铁路局”。
- 常见后果：类型、地址、坐标都可能被带偏。
- 质检动作：区分机构主体与服务/管理主体。

### FP04 把子点位当成整体 POI

- 表现：证据只证明航站楼、游客中心、售票处、景区内单景点、站前广场。
- 常见后果：整体名称、类型、坐标被误判通过。
- 质检动作：判断证据是否能外推到整体实体；不能外推时判 `manual` 或 `fail`。

### FP05 把关联交通设施当成目标 POI

- 表现：把机场地铁站、火车站周边枢纽、停车场、公交站当作目标 POI。
- 常见后果：坐标和地址常常看似接近，但实体错了。
- 质检动作：验证实体边界与功能。

### FP06 只证存在，不证字段

- 表现：证据只能说明“这个地方有这个 POI”。
- 常见后果：上游顺带把类型、地址、坐标一并判通过。
- 质检动作：逐字段卡证据，不允许整体兜底。

### FP07 地址拿了办公地址或邮寄地址

- 表现：证据地址属于办公室、机关楼、管理处，而非 POI 主体。
- 常见后果：政府机构、景区最常见。
- 质检动作：确认地址语义，是办公地址、管理地址还是对外入口地址。

### FP08 坐标拿了附近点

- 表现：地图点位接近，但落在道路、停车场、地铁站、周边商业体。
- 常见后果：上游误认为“差不多就算对”。
- 质检动作：按 POI 类型检查容差和定位语义，不能只看距离。

### FP09 引用链不独立

- 表现：多个证据其实都转自同一篇原文或同一平台数据。
- 常见后果：上游误以为“多源一致”。
- 质检动作：折叠成单一来源重新判断。

### FP10 把规划/在建当成现状

- 表现：证据来自可研、公示、奠基、施工新闻。
- 常见后果：存在性被误判为已建成/已投入使用。
- 质检动作：查是否有“已启用/已开放/当前运营”级别的现状证据。

### FP11 5A 资质状态过期

- 表现：景区曾是 5A，后续降级、摘牌、整改，证据未更新。
- 常见后果：类型误判通过。
- 质检动作：优先看最新文旅部/文旅厅局名单与通告。

### FP12 车站新老站混淆

- 表现：老站仍存在，新站投入使用，名称接近。
- 常见后果：地址、坐标、存在性混乱。
- 质检动作：确认是否为同一站、同一站房或不同客运实体。

### FP13 机场整体与航站楼混淆

- 表现：证据描述 T1/T2/T3，但上游输出机场整体结论。
- 常见后果：坐标与地址表面正确，实际语义错位。
- 质检动作：明确证据支撑的是整体还是子设施。

### FP14 景区名与核心景点名混淆

- 表现：景区整体名、核心景点名、游客中心名混用。
- 常见后果：名称和坐标都容易误判。
- 质检动作：看官方 5A 名录中的正式景区名。

### FP15 行政中心与具体机关混淆

- 表现：某市行政中心、政务服务中心、区政府机关大楼被混为同一实体。
- 常见后果：政府类名称、地址、坐标混乱。
- 质检动作：确认“地点建筑”和“机关主体”是否是同一核实对象。

## Strong Fail Signals

以下情况通常直接支持 `fail`：

- 高等级证据明确指向另一座城市或另一实体。
- 上游把管理机构、子点位、关联交通设施当成目标 POI。
- 上游输出精确坐标，但证据只能支撑模糊范围或近邻点位。
- 上游使用过期证据支持“当前状态”。

## Typical Manual Signals

以下情况更适合 `manual`：

- 新旧名称并存，但有可能是同一实体。
- 地址和坐标都接近，但证据无法确认是主体还是入口。
- 景区类证据只能确认游客中心，无法确认整体 POI 语义。
- 多条证据都较可信，但相互冲突且缺少最终权威解释。
